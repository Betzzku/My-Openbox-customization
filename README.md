## File Manager — Thunar

**Installed:**
```bash
sudo pacman -S thunar thunar-volman thunar-archive-plugin gvfs tumbler
```

**Autostart** (`~/.config/openbox/autostart`):
```bash
thunar --daemon &
```

**Keybind** (`~/.config/openbox/rc.xml`):
```xml
<keybind key="W-f">
  <action name="Execute">
    <command>thunar</command>
  </action>
</keybind>
```

**Custom right-click actions:** configured via Thunar's own `Edit → Configure Custom Actions...` (not stored in dotfiles, lives in Thunar's own config).

## Polybar — Audio Module

**Module config** (`~/.config/polybar/config.ini`, `[module/pulseaudio]`):
```ini
[module/pulseaudio]
type = internal/pulseaudio
label-volume = audio %percentage%%
label-muted = muted

scroll-up = pamixer --increase 5 && ~/.config/polybar/scripts/volume-notify.sh
scroll-down = pamixer --decrease 5 && ~/.config/polybar/scripts/volume-notify.sh
click-left = pamixer --toggle-mute && ~/.config/polybar/scripts/volume-notify.sh
right-click = pavucontrol
```

**Notification script** (`~/.config/polybar/scripts/audio-menu.sh`):
```bash
#!/bin/bash

vol=$(pamixer --get-volume)
muted=$(pamixer --get-mute)

if [ "$muted" = "true" ]; then
  notify-send -h string:x-dunst-stack-tag:volume "Volume" "Muted"
else
  notify-send -h int:value:"$vol" -h string:x-dunst-stack-tag:volume "Volume" "${vol}%"
fi
```

Notes:
- Uses `dunst` (already running as notification daemon) to render the volume bar/percentage
- `x-dunst-stack-tag:volume` makes repeated scrolls replace the same notification instead of stacking
- Considered SwayOSD, but it requires Wayland/wlroots (layer-shell protocol) and does **not** render under X11/Openbox — abandoned in favor of dunst
- Considered a yad popup slider as an alternative — works but was replaced by the simpler dunst notification approach

## Screenshots — maim

**Installed:**
```bash
sudo pacman -S maim libnotify
```

**Keybind** (`~/.config/openbox/rc.xml`, Super+Shift+S):
```xml
<keybind key="W-S-s">
  <action name="Execute">
    <command><![CDATA[bash -c "maim -s ~/Pictures/Screenshots/screenshot_$(date +%Y-%m-%d_%H-%M-%S).png && notify-send 'Screenshot saved!'"]]></command>
  </action>
</keybind>
```

Notes:
- Each screenshot gets a unique timestamped filename for example screenshot-2026-02-02.jpg
- Selection mode (`-s`): click-drag to choose a region
- Switched from Flameshot after it failed — Flameshot's newer versions require an XDG Desktop Portal, which isn't available on a bare Openbox/X11 setup, causing a permanent timeout
- **Important:** ampersands (`&&`) inside `<command>` must be escaped as `&amp;&amp;` in raw XML, or wrapped in `<![CDATA[...]]>` as shown above — a raw `&&` will break the whole rc.xml parse

## Bootloader — Limine (CachyOS)

**Config file:** `/boot/limine.conf`

Notes on structure:
- Global options (timeout, default_entry, wallpaper, palette) — **safe to hand-edit directly**
- Entries under `/+CachyOS` (kernel paths, checksums, cmdline) — **auto-generated by `limine-entry-tool`**, do not hand-edit; changes will be overwritten on kernel update
- Persistent kernel cmdline changes belong in `/etc/kernel/cmdline`, then regenerate via `limine-mkinitcpio` (or CachyOS's equivalent hook)

Customizations discussed:
- Lowering `timeout` from 30s
- Fixing `default_entry` to point at the actual CachyOS kernel entry (was pointing at "Other systems and bootloaders")
- Custom `wallpaper:` pointing to a file placed in `/boot/`
- Optional: `term_palette` / `term_foreground` adjusted to match the polybar green/mono scheme (theme currently uses `cachyos-limine-theme`, whose `a6e3a1` green already closely matches the polybar primary color)

## Lock Screen — betterlockscreen (replacing slock)

**Installed (AUR):**
```bash
sudo pacman -S imagemagick libxft libxinerama
yay -S betterlockscreen
```

**Config location:** `~/.config/betterlockscreen/betterlockscreenrc`
*(migrated from the old `~/.config/betterlockscreenrc` path automatically)*

**Config contents:**
```bash
# Blur/dim
blur_level=5
lock_effect="blur"

# Colors matching polybar palette
text_color="c8d6c8"
insidewrongcolor="f38ba8"
insidevercolor="a6e3a1"
ringcolor="a6e3a1"
ringvercolor="a6e3a1"
ringwrongcolor="f38ba8"

# Font
font="Noto Sans Mono"

# Password ring position (centered on 1920x1080)
indicator_position="x+960 y+540"
```

**Generate lock cache from wallpaper** (must re-run after wallpaper/config changes):
```bash
betterlockscreen -u ~/Pictures/anime-background.jpg
```

**Keybind** (`~/.config/openbox/rc.xml`, Super+L):
```xml
<keybind key="W-l">
  <action name="Execute">
    <command>betterlockscreen -l</command>
  </action>
</keybind>
```

Notes:
- Replaced `slock` (which was already installed) because slock has no config file — customization would've required editing its source `config.h` and recompiling from scratch, and it can't show a wallpaper at all (solid color only)
- `betterlockscreen` wraps `i3lock-color` under the hood
- Blur wasn't applying at first — still being debugged as of the last session; if it recurs, check `betterlockscreen --help | grep -i blur` for the exact flag/config-key syntax your installed version expects, since option names have changed across forks/versions
